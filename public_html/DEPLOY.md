# Chocolandia.by — Руководство по развёртыванию

## Что это за сайт?

Статическое веб-приложение (SPA) на чистом HTML + CSS + JS.
- Никаких зависимостей, никакого Node.js, никакой сборки
- Работает на любом Apache-хостинге, включая Hoster.by (домен .by)
- Полная поддержка красивых URL через `.htaccess`

---

## Структура файлов

```
public_html/
├── .htaccess                  ← Apache routing rules (обязательно!)
├── index.html                 ← SPA-оболочка
├── app.js                     ← Роутер + все страницы
├── styles.css                 ← Весь дизайн
├── data/
│   └── products.json          ← База данных товаров (редактируйте здесь)
└── assets/
    └── images/
        ├── hero_banner.png
        ├── strawberry_chocolate.png
        ├── dubai_chocolate.png
        ├── truffles.png
        ├── gift_box.png
        └── easter_collection.png
```

## Шаг 1: Подготовка к загрузке

1. Убедитесь, что папка `public_html/` содержит все файлы (см. выше)
2. Убедитесь, что файл `.htaccess` присутствует (он скрытый — он начинается с точки)
3. Если нужно добавить свои фотографии товаров — замените файлы в `assets/images/`

---

## Шаг 2: Загрузка через FTP

### Способ 1: FileZilla (рекомендуется)

1. Откройте FileZilla
2. Подключитесь к хостингу Hoster.by:
   - **Хост:** ftp.chocolandia.by (или IP из письма хостинга)
   - **Имя пользователя:** (ваш логин)
   - **Пароль:** (ваш пароль)
   - **Порт:** 21
3. Перейдите в корневую папку сайта (обычно `public_html/` или `www/`)
4. Загрузите **всё содержимое** папки `public_html/` в корень сайта

> ⚠️ **Важно:** Загружайте содержимое папки, а НЕ саму папку `public_html`.
> Файл `index.html` должен лежать напрямую в корне сайта на хостинге.

### Способ 2: Файловый менеджер в панели Hoster.by

1. Войдите в панель управления Hoster.by
2. Откройте «Файловый менеджер»
3. Перейдите в `/public_html/` (или `/www/`)
4. Загрузите все файлы и папки

---

## Шаг 3: Проверка .htaccess

После загрузки убедитесь, что `.htaccess` работает:

1. Откройте `https://chocolandia.by/`
2. Перейдите в коллекцию, например: `https://chocolandia.by/collections/strawberry`
3. Нажмите **F5** (обновить страницу)
4. Если страница загрузилась без ошибки 404 — всё работает ✓

Если появляется 404, значит `.htaccess` не загружен или mod_rewrite не включён. Обратитесь в поддержку Hoster.by.

## Как добавить или изменить товары

Все товары хранятся в файле `data/products.json`. Откройте его в любом текстовом редакторе.

### Структура коллекции:
```json
{
  "id": "strawberry",
  "slug": "strawberry",
  "name": "Клубника в шоколаде",
  "description": "Описание коллекции",
  "image": "assets/images/strawberry_chocolate.png",
  "badge": "Хит продаж",
  "featured": true,
  "sortOrder": 1
}
```

### Структура товара:
```json
{
  "id": "strawberry-dark-box",
  "slug": "strawberry-dark-box",
  "collectionId": "strawberry",
  "name": "Клубника в тёмном шоколаде",
  "description": "Описание товара",
  "price": 24.90,
  "unit": "BYN",
  "weight": "200 г",
  "image": "assets/images/strawberry_chocolate.png",
  "badge": "Хит",
  "inStock": true,
  "tastingNotes": ["Нотка 1", "Нотка 2"],
  "ingredients": "Клубника, шоколад..."
}
```

### Правила для `slug`:
- Только латинские буквы, цифры и дефисы
- Без пробелов и кириллицы
- Должен быть уникальным
- Пример: `dubai-pistachio-dark`

## Как добавить настоящие фотографии

1. Сделайте фото товара (рекомендуемое разрешение: **1200×1500 px**, соотношение 4:5)
2. Сохраните в формате **JPG** или **WebP** для оптимального размера
3. Поместите файл в папку `assets/images/`
4. В файле `products.json` укажите путь: `"image": "assets/images/ваш-файл.jpg"`

## Проблемы и решения

| Проблема | Решение |
|----------|---------|
| 404 при обновлении страницы | Проверьте, загружен ли `.htaccess` и включён ли mod_rewrite |
| Изображения не отображаются | Проверьте путь в `products.json` — он должен быть `assets/images/имя.png` |
| Сайт открывается, но товары не загружаются | Проверьте файл `data/products.json` на валидность JSON |
| Шрифты не загружаются | Нужен интернет — шрифты подключены через Google Fonts CDN |

## Локальное тестирование

Для тестирования на своём компьютере без хостинга:

```bash
# Перейдите в папку public_html
cd "d:\vibecoding\Chcolandia v3 latest\public_html"

# Запустите Python-сервер
python -m http.server 8080

# Откройте в браузере
# http://localhost:8080/
```

> ⚠️ При локальном тестировании через `python -m http.server` красивые URL типа `/collections/strawberry` при обновлении страницы дадут 404 — это нормально. На реальном хостинге с `.htaccess` это работает корректно.
