# Chocolandia.by — Шоколад ручной работы 🍫

Высокопроизводительный Single Page Application (SPA) для онлайн-каталога крафтового шоколада. Сайт спроектирован для максимальной скорости загрузки, плавного пользовательского интерфейса (UX) и безопасности заказов.

## 🚀 Технологический стек

*   **Frontend архитектура:** Нативный (Vanilla) JavaScript. Архитектура SPA без сторонних фреймворков.
*   **Стилизация (CSS):** Vanilla CSS с использованием современных CSS Variables для консистентной дизайн-системы, Flexbox/Grid для адаптивной верстки. Мобильная адаптация через Media Queries.
*   **Хранение данных:** Статичная JSON-база (`data/products.json`), которая кэшируется браузером на клиенте. Корзина хранится в локальной памяти (`localStorage`).
*   **Backend интеграция:** Серверный PHP-скрипт (`api/order.php`) в качестве безопасного промежуточного узла (Bridge). Сервер скрывает токен от публичного веба и по API стучится в Telegram Bot API.
*   **Сервер и маршрутизация:** Apache Server с кастомным `.htaccess` для перенаправления URL (поддержка "красивых ссылок" в SPA) и форсирования UTF-8.

## 📁 Структура проекта

```text
public_html/
├── index.html          # Главная и единственная точка входа (Shell)
├── styles.css          # Дизайн-система и стили всех компонентов
├── app.js              # Бизнес-логика, рендеринг, роутер, корзина
├── .htaccess           # Настройки Apache: роутинг, кэш, gzip, utf-8
├── data/
│   └── products.json   # Вся база товаров и категорий!
└── api/
    └── order.php       # PHP бэкенд для безопасной отправки заказа
```

> **Важно по безопасности:** Файл настроек бота `telegram_config.php` хранится на сервере на один уровень **выше** `public_html/`, чтобы сервер не отдавал токен при попытке прямого перехода.

## 🛠 Руководство: Управление товарами

Сайт полностью автономен и не требует базы данных SQL или сложной админки. Весь ассортимент, цены, категории и описания редактируются в одном файле: `public_html/data/products.json`.

Достаточно открыть его в любом текстовом редакторе (VS Code, cPanel File Manager). После сохранения изменений — сайт обновится автоматически!

### Как устроен файл `products.json`
Файл состоит из двух массивов:
1.  **`"collections"`** — Категории (коллекции).
2.  **`"products"`** — Каталог товаров.

### Добавление нового товара
Скопируйте любой существующий блок с товаром и вставьте через запятую.
Пример идеального товара:

```json
{
  "id": "truffle-mint",
  "slug": "truffle-mint",
  "collectionId": "truffles",
  "name": "Трюфель Мята",
  "description": "Освежающий трюфель с натуральной перечной мятой.",
  "price": 12.50,
  "unit": "BYN",
  "weight": "80 г",
  "image": "/assets/images/mint_truffle.png",
  "badge": "Новинка",
  "inStock": true,
  "tastingNotes": ["Тёмный шоколад", "Мята"],
  "ingredients": "Шоколад, сливки, эфирное масло мяты"
}
```

### Основные правила редактирования (Шпаргалка):
| Параметр | Пояснение |
| :--- | :--- |
| **`id` / `slug`** | Уникальные идентификаторы. Пишем только латиницей, без пробелов (имя-товара). |
| **`collectionId`** | К какой категории относится товар (например, `strawberry`, `truffles`, `gifts`). Берутся из раздела `"collections"`. |
| **`price`** | Цифра! Используется **точка** для копеек, кавычки ставить **не нужно** (Пример: `29.90`, а не `"29,90"`). |
| **`badge`** | "Флажок" товара (`"Хит продаж"`, `"Новинка"`). Если флажок не нужен, пишем обязательно без кавычек: `"badge": null`. |
| **`image`** | Путь к картинке. Сначала загрузите изображение в `/assets/images/`, затем пропишите путь. |
| **`inStock`** | В наличии ли товар? (`true` — да, `false` — временно нет). |

### ⚠️ Правила синтаксиса JSON
- Все слова (ключи и значения текста) оборачиваются в **двойные кавычки** `" "`. Двойные кавычки внутри текста использовать нельзя.
- Между свойствами ставится **запятая** `,`.
- **ВНИМАНИЕ:** В самой последней строке внутри блока товара (обычно это `"ingredients"`) запятой в конце быть не должно!
